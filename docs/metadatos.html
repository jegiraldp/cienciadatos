Notas de Clase: XML y JSON

¿Qué es un metadato?

Un metadato es información que describe otros datos. Su función principal es facilitar la comprensión, organización, búsqueda y gestión de la información. Los metadatos no suelen contener el dato principal en sí, sino características asociadas a él, como su origen, formato, fecha de creación, autor o estructura.

En el contexto de tecnologías como XML y JSON, los metadatos pueden describir la estructura de los datos, su semántica o las reglas que deben cumplir, permitiendo la interoperabilidad entre sistemas.

XML (eXtensible Markup Language)

Definición

XML es un lenguaje de marcado extensible diseñado para almacenar y transportar datos. Se basa en una estructura jerárquica de etiquetas definidas por el usuario, lo que permite describir la información de manera clara y estructurada.

Aplicaciones

  • Intercambio de información entre sistemas heterogéneos.
  • Configuración de aplicaciones y servicios.
  • Representación de documentos estructurados.
  • Servicios web (SOAP).

Herramientas y editores

  • Visual Studio Code
  • Oxygen XML Editor
  • Notepad++
  • XMLSpy

Ejemplos de código XML

<?xml version="1.0" encoding="UTF-8"?>
<persona>
  <nombre>Ana</nombre>
  <edad>25</edad>
</persona>
<?xml version="1.0" encoding="UTF-8"?>
<libro>
  <titulo>Programación</titulo>
  <autor>Juan Pérez</autor>
</libro>
<?xml version="1.0" encoding="UTF-8"?>
<curso>
  <nombre>Bases de Datos</nombre>
  <creditos>3</creditos>
</curso>
<?xml version="1.0" encoding="UTF-8"?>
<producto id="101">
  <nombre>Laptop</nombre>
  <precio>2500</precio>
</producto>
<?xml version="1.0" encoding="UTF-8"?>
<universidad>
  <facultad>Ingeniería</facultad>
</universidad>

Comparación entre XML y JSON

XML y JSON son formatos ampliamente utilizados para el intercambio de datos, pero presentan diferencias importantes en cuanto a estructura, complejidad y casos de uso.

  • Estructura: XML utiliza etiquetas jerárquicas; JSON usa pares clave-valor.
  • Legibilidad: JSON suele ser más compacto y fácil de leer.
  • Validación: XML cuenta con DTD y XSD; JSON utiliza JSON Schema.
  • Uso común: XML es frecuente en sistemas legados; JSON domina en APIs modernas.

  <edad>25</edad>

<persona>
  <nombre>Ana<nombre>  <!-- Falta barra de cierre -->
</persona>

JSON (JavaScript Object Notation)

Definición

JSON es un formato ligero de intercambio de datos basado en texto, fácil de leer y escribir para los humanos y sencillo de procesar para las máquinas. Utiliza una estructura de pares clave-valor y arreglos.

Aplicaciones

  • Comunicación entre aplicaciones web y servidores (APIs REST).
  • Almacenamiento de configuraciones.
  • Intercambio de datos en aplicaciones móviles.
  • Persistencia de datos en bases NoSQL.

Herramientas y editores

  • Visual Studio Code
  • Sublime Text
  • Postman
  • JSON Formatter Online

Ejemplos de código JSON

{
  "nombre": "Ana",
  "edad": 25
}
{
  "titulo": "Programación",
  "autor": "Juan Pérez"
}
{
  "curso": "Bases de Datos",
  "creditos": 3
}
{
  "producto": {
    "nombre": "Laptop",
    "precio": 2500
  }
}

Errores comunes en JSON

{
  "nombre": "Ana",
  "edad": 25,  
}  // Coma final no permitida
{
  nombre: "Ana",
  edad: 25
}  // Claves sin comillas
{
  "universidad": "Instituto Tecnológico",
  "facultad": "Ingeniería"
}
